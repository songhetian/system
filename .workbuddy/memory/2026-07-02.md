## design-an-interface: 全套 Zod Schema + TypeScript 类型生成

产出 `src/types/` 目录，16 个文件，Schema-First 推导模式：

- `common.ts`：ApiResponse<T>、PaginatedData<T>、auditMixin、softDeleteMixin、基础校验 Schema
- `auth.ts` / `org.ts` / `employee.ts` / `permissions.ts`：组织权限域
- `schedules.ts` / `attendance.ts` / `leave.ts`：排班考勤假期域（含 .superRefine 业务规则）
- `salary.ts`：薪资二级密码 + 脱敏列表/详情推导
- `expense.ts` / `training.ts` / `kb.ts`：报销培训知识库
- `workflow.ts`：BPMN-lite，z.discriminatedUnion 6 种节点辨析联合
- `messages.ts` / `excel.ts`：消息公告 + Excel 导入导出
- `index.ts`：根 barrel 统一导出

核心特征：
- 零手写 interface/type，全部 `z.infer`
- Create → .partial() → Update, Base → .omit()/.extend() → ListItem/FullDetail
- 跨字段校验用 .superRefine（请假日期、轮班周期、审批流结构）
- 递归组织树用 z.lazy()，6 种审批节点用 z.discriminatedUnion('type')
